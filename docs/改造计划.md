# 改造计划：从 Paper Trading 到真实自动交易

## 目标

将现有的 Polymarket BTC 15m 模拟交易项目改造为**真实自动交易 Bot**，使用官方 `py-clob-client` SDK 实现链上下单。

## 背景分析

### 现有代码现状
- `bot.py` 有 2434 行，包含 3 种模式：
  - `TradingBot`（旧版 live 模式，约 L382-L891）：**自写的假 API 客户端**，认证方式错误（Bearer token 而非 HMAC 签名），`place_order` 接口参数不对，**完全不可用**
  - `PaperReplayBot`（回放模式，约 L894-L1197）：离线回放历史数据
  - `ContinuousPaperTradingBot`（模拟交易模式，约 L1199-L2397）：**当前运行的核心**，代码质量最好，已实现完整的市场扫描、AI 决策、仓位管理、止盈/到期离场逻辑
- **核心问题**：现有的 `PolymarketClient` 类（L383-L451）没有使用官方 SDK，不会 EIP-712 签名，不支持 Proxy Wallet，**无法真实下单**。

### 改造策略
**最高效的方案**：在 `ContinuousPaperTradingBot` 基础上改造，仅替换"开仓/平仓"的执行层为真实 SDK 下单，保留现有的：
- ✅ 市场扫描逻辑（`build_today_slot_slugs` → `get_today_market_snapshots`）
- ✅ AI 决策引擎（`AIDecisionEngine` + OpenClaw 外部信号）
- ✅ 仓位管理和状态持久化
- ✅ 前端监控面板
- ✅ 归档和启停脚本

### 阶段 2：新建真实交易执行层（核心改造）

#### [NEW] [live_trader.py](file:///Users/sonic/Desktop/%E7%BD%91%E9%A1%B5%E8%AE%BE%E8%AE%A1/polymarket_web/live_trader.py)

新建一个独立模块封装官方 SDK 的下单逻辑，与现有代码完全解耦：

```python
# live_trader.py - 真实交易执行层
# 封装 py-clob-client SDK，提供简洁的下单/取消/查询接口

class LiveTrader:
    """封装 Polymarket 官方 SDK 的真实交易执行器"""
    
    def __init__(self, private_key, funder_address, 
                 signature_type=1, api_creds=None, dry_run=False):
        """
        初始化 SDK 客户端
        - private_key: 钱包私钥
        - funder_address: 充值钱包地址（Proxy Wallet 的 funder）
        - signature_type: 0=EOA, 1=Proxy, 2=MagicLink
        - api_creds: 可选，手动提供的 API 凭证（跳过 derive）
        - dry_run: 如果 True，只打印不下单
        """
    
    def buy(self, token_id, price, size, tick_size="0.01", neg_risk=False):
        """
        买入指定 outcome 代币
        - token_id: 市场的 clobTokenIds 中对应的 ID
        - price: 限价（0.01-0.99）
        - size: 买入金额（USDC）
        - 返回: order_id 或 None
        """
    
    def sell(self, token_id, price, size, tick_size="0.01", neg_risk=False):
        """卖出（平仓）"""
    
    def get_open_orders(self):
        """查询当前挂单"""
    
    def cancel_order(self, order_id):
        """取消指定挂单"""
    
    def cancel_all_orders(self):
        """取消所有挂单"""
    
    def get_balances(self):
        """查询链上 USDC.e 余额"""
```

**关键实现细节**：
- SDK 初始化流程：`ClobClient(host, key, chain_id)` → `create_or_derive_api_creds()` → `set_api_creds()`
- 由于用户已有手动创建的 API 凭证，优先尝试用已有凭证初始化，失败再 derive
- 所有下单操作通过 `create_and_post_order()` 完成
- 包装 `dry_run` 模式，只打印下单参数不实际提交
- 内置重试和异常处理

1. **新增配置变量**（在现有配置部分添加）：
   ```python

   POLYMARKET_SIGNATURE_TYPE = int(os.getenv("POLYMARKET_SIGNATURE_TYPE", "1"))

   ```

2. **新建 `LiveTradingBot` 类**：
   - 继承 `ContinuousPaperTradingBot` 的大部分逻辑
   - 重写 `open_position()` → 调用 `LiveTrader.buy()`
   - 重写 `close_position()` → 调用 `LiveTrader.sell()`
   - 重写 `refresh_open_positions()` → 额外从链上同步真实持仓状态

3. **修改 `main()` 入口**：
   ```python
   if TRADING_MODE == "live":
       bot = LiveTradingBot()
       await bot.run()
   ```

**核心设计**：`LiveTradingBot` 与 `ContinuousPaperTradingBot` 共享 90% 的代码（市场扫描、AI 决策、状态管理）。唯一不同的是执行层——纸上交易只修改本地余额，真实交易调用 SDK 下单。

## 开放问题

> [!IMPORTANT]
> **问题 1**：你的 AI API Key 是什么？之前的 `.env` 中是占位符。需要一个有效的 OpenAI / Claude / 其他兼容 API 的 Key 才能让 AI 决策正常工作。如果没有 AI Key，Bot 会回退到简单的"现价 vs 今日开盘"规则策略。

> [!IMPORTANT]
> **问题 2**：确认你的钱包里有多少 USDC.e？可以在 [Polygonscan](https://polygonscan.com/address/0xebB80D98Ba64ED39b0de71ceE18E101Ddbb6bfa9) 查看。需要至少有 LIVE_BET_AMOUNT（默认 1 USDC）+ 少量 POL（gas）才能交易。

> [!WARNING]
> **问题 3**：Python 版本是 3.14.3。`py-clob-client` 官方最低支持 3.9，但 3.14 比较新，可能存在兼容性问题。如果安装失败，可能需要创建一个 Python 3.12 的虚拟环境。
